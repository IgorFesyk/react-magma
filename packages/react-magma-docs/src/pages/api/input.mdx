Input fields are used in a number of ways for both collecting and displaying data from the user.

## Basic Usage

The label/input association will be handled via the passed in `id` if one is supplied or via an auto-generated `id` if not supplied.

```.jsx
    <>
      <Input
          labelText='Label' />
      <Input
          id="myCustomId"
          labelText='Label' />
    </>
```

## Password

When the `type` prop is set to `password`, the input field that is rendered also have the `type` set to password, meaning that the text is obscured so that it cannot be read, usually by replacing each character with a symbol such as the asterisk ("\*") or a dot ("â€¢").

By default, users can show the password, by pressing a toggle button. To prevent users from showing the password, the optional boolean `hidePasswordMaskButton` can be used.

The following option props can be used to override various text strings that appear on the UI or to users using assistive technology: `hiddenPasswordAnnounceText`, `hidePasswordButtonAriaLabel`, `hidePasswordButtonText`, `shownPasswordAnnounceText`, `showPasswordButtonAriaLabel`, `showPasswordButtonText`.

```.jsx
  <>
    <Input
      labelText='Password'
      type='password' />
    <Input
      hidePasswordMaskButton
      labelText='Password With No Unmask Button'
      type='password' />
  </>
```

## Number

```.jsx
    <Input
        labelText='Number'
        type='number' />
```

## Disabled

```.jsx
    <Input
        labelText='Disabled'
        disabled />
```

## Value

The `value` of an input can be set using the `value` prop.

```.jsx
    <Input
        labelText='Default value'
        value='abc123' />
```

## Placeholder

The `placeholder` text of an input can be set using the `placeholder` prop.
Placeholder text should be used to provide supplemental information about the input field. It should not be
relied upon to take the place of the label text.

```.jsx
    <Input
        labelText='Email Address'
        value='yourname@example.com' />
```

## Error Message

If an input has an `errorMessage`, the input will be styled to highlight it's error state and the error message will appear below the input field.
If an error message is present, it will replace the helper text.

```.jsx
    <Input
        errorMessage='Please correct this error'
        helperMessage='Helper text'
        labelText='Error Message' />
```

## Helper Message

The `helperMessage` appears underneath the input field. It will not appear if an error message is present.

```.jsx
    <Input
        helperMessage='Helper text goes here'
        labelText='Helper Message' />
```

## Help Link

Some inputs require a bit of explanation to the user. Passing in a function to the `onHelpLinkClick` prop, will
display a help link to the right of the input field. The text can be customized using the optional `helpLinkText` prop.

```.jsx
   <Input
      labelText='Help Link'
      onHelpLinkClick={()=> {alert('Help link clicked!')}} />
```

## Input With Icon

An `icon` may be included with the input, by passing in the `icon` prop. An aria-label may be added using the `iconAriaLabel` prop.

Functionality can be added to the icon by using the `onIconClick` and `onIconKeyDown` props. All clickable input icons will appear on the right.

The `iconPosition` position prop is optional and takes the values `left` and `right`, with `left` being the default for non-clickable icons.

```.jsx
    <>
      <Input
        labelText='Input with Icon on Left'
        icon={<EnvelopeOIcon />}
        type="email" />

      <Input
        iconPosition='right'
        labelText='Input with Icon on Right'
        icon={<Search2Icon />} />

      <Input
        labelText='Input with Clickable Icon on Right'
        iconAriaLabel="Calendar"
        onIconClick={()=>{alert('Icon clicked')}}
        icon={<CalendarIcon />} />
    </>
```

## Input Size

The `inputSize` prop can be used to control the input size. Options are `medium` and `large`, with `medium` being the default. Input size does not affect multiline textareas.

```.jsx
    <>
      <Input
        iconPosition='right'
        labelText='Medium Input'
        icon={<Search2Icon />} />
      <Input
        iconPosition='right'
        inputSize='large'
        labelText='Large Input'
        icon={<Search2Icon />} />
      <Input
          errorMessage='Error'
          labelText='Medium Input With Error' />
      <Input
          errorMessage='Error'
          inputSize='large'
          labelText='Large Input With Error' />
      <Input
          onHelpLinkClick={()=> {alert('Help link clicked!')}}
          labelText='Medium Input With Help Link' />
      <Input
          onHelpLinkClick={()=> {alert('Help link clicked!')}}
          inputSize='large'
          labelText='Large Input With Help Link' />
    </>
```

## IsLoading

The `isLoading` boolean prop will determine if a loading animation is shown. The loading animation will replace any icon displayed on the right.

```.jsx
    <>
      <Input labelText='Loading' isLoading />
      <Input labelText='Loading Large' inputSize='large' isLoading />
    </>
```

## Inverse

Inverse is an optional boolean prop, that may be passed in when inputs are to be displayed on a dark background.

```.jsx
    <div style={{ background : '#00263E', padding: '20px' }}>
      <Input
          id='inputInverse'
          inverse
          labelText='Inverse Input'
          placeholder='Type in here' />

      <Input
          helperMessage='Helper text'
          inverse
          labelText='Inverse Input with Helper Text' />

      <Input
          errorMessage='Please correct your error'
          inverse
          labelText='Inverse Input with Error Message' />
      <Input
          inverse
          labelText='Help Link'
          onHelpLinkClick={()=> {alert('Help link clicked!')}} />
    </div>
```

## Input With Hidden Label

The `labelVisuallyHidden` prop can be used to visually hide the text label, while still making the text available to screen readers.

```.jsx
    <Input
      iconPosition='right'
      labelVisuallyHidden
      labelText='Search'
      placeholder='Search'
      icon={<Search2Icon />} />
```

## Multi-line (textarea)

The optional `multiline` boolean prop will cause the component to render a textarea, rather than a text input.

```.jsx
    <Input
        id='textarea'
        labelText='Multi-line Textarea'
        multiline
        placeholder='Type in here' />
```

## Change events

```.jsx
  class ChangedInput extends React.Component {
    constructor(props) {
      super(props)
      this.state = {
        message: ''
      };

      this.changeValueMessage = this.changeValueMessage.bind(this);
    }

    changeValueMessage(event) {
      this.setState({message: event.target.value})
    }

    render() {
      return <div>
        <p><strong>Changed Value: </strong> <span id="changedValue">{this.state.message}</span></p>
        <Input
          labelText='Test Value'
          value=''
          onChange={this.changeValueMessage} />
      </div>
    }
  }
```

## Focus events

```.jsx
  class FocusedInput extends React.Component {
    constructor(props) {
      super(props)
      this.state = {
        focused: false,
        message: 'No'
      };

      this.changeFocusedMessage = this.changeFocusedMessage.bind(this);
    }

    changeFocusedMessage() {
      const focused = !this.state.focused;

      this.setState(state => ({
        focused,
        message: focused ? 'Yes' : 'No',
      }));
    }

    render() {
      return <div>
        <p><strong>Focused: </strong> <span id="focused">{this.state.message}</span></p>
        <Input
          id='inputFocused'
          labelText='Focused?'
          onFocus={this.changeFocusedMessage}
          onBlur={this.changeFocusedMessage} />
      </div>
    }
  }
```

## Forward ref

Using React's `forwardRef` feature you can gain access to the reference of the internal input.

```.jsx
  class Example extends React.Component {
      constructor(props) {
        super(props)
        this.inputRef = React.createRef();
      }

      render() {
        return <div>
        <Input
          ref={this.inputRef}
          labelText='Input to be focused' />

        <Button
          onClick={() => {
            this.inputRef.current.focus();
          }}>Click to focus input
        </Button>
        </div>
      }
    }
```

## Custom styles

Custom styles can be passed into the Input component. The `containerStyle` property will apply to the container.
Additional `labelStyle` and `inputStyle` properties are available to style the respective elements. Please use discretion when adding custom styles.

```.jsx
  <div>
    <Input
        containerStyle={{ 'border': '1px dotted purple', 'padding': '5px'}}
        inputStyle={{ 'borderRadius': '0'}}
        labelStyle={{ 'color': 'brown', 'fontStyle': 'italic'}}
        labelText='Custom Styles for input' />

    <Input
        containerStyle={{ 'border': '1px dotted purple', 'padding': '5px'}}
        inputStyle={{ 'width': '200px', 'height': '50px'}}
        labelStyle={{ 'color': 'brown', 'fontStyle': 'italic'}}
        labelText='Custom Styles for textarea (multiline)'
        multiline />
  </div>
```

## Testing

Passing in the `testId` prop will add the `data-testid` attribute to the input element for easier querying in tests.

```html
<div>
  <label for="input1">Label</label>
  <div>
    <input
      id="input1"
      data-testid="test-id"
      placeholder="This is a placeholder"
      type="text"
      value="Default Value"
    />
  </div>
</div>
```
