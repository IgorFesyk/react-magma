# Breakpoints Container

The `BreakpointsContainer` component can be used to hide/show content at different screen sizes. Breakpoint values are defined in the theme, and can be overridden at the component level.

By placing multiple `Breakpoint` components inside the container, you can define multiple versions of the content to be displayed.

Content will be shown for the specified breakpoint, up to the next largest breakpoint.

This component uses the <Link to="/api/hide-at-breakpoint/">Hide at Breakpoint</Link> component.

## Basic Usage

```.jsx
    <BreakpointsContainer implementation='js'>
         <Breakpoint screenSize="xs">
            <Card background={magma.colors.pop04}>
                <CardBody><strong>Extra-Small: </strong>This will be shown from 0 pixels, up to 599px.</CardBody>
            </Card>
        </Breakpoint>


        <Breakpoint screenSize="small">
            <Card isInverse background={magma.colors.foundation02}>
                <CardBody><strong>Small: </strong>This will be shown from 600px pixels up to 767px.</CardBody>
            </Card>
        </Breakpoint>

        <Breakpoint screenSize="medium">
            <Card isInverse background={magma.colors.pop01}>
                <CardBody><strong>Medium: </strong>This text will be visible when the browser width is 768px and up to 1023px.</CardBody>
            </Card>
        </Breakpoint>

        <Breakpoint screenSize="large">
            <Card isInverse background={magma.colors.success01}>
                <CardBody><strong>Large: </strong>This text will be visible when the browser width is 1024px and to 1199px.</CardBody>
            </Card>
        </Breakpoint>

         <Breakpoint screenSize="xl">
            <Card isInverse background={magma.colors.danger}>
                <CardBody><strong>Extra-Large: </strong>This text will be visible when the browser width is 1200px and greater.</CardBody>
            </Card>
        </Breakpoint>

        <br/>
        <Card>
            <CardBody>This text will always be visible, as it is not part of a breakpoint.</CardBody>
        </Card>
    </BreakpointsContainer>
```

## Using Fewer Breakpoints

It is not neccessary to use all of the breakpoints. If only two are used, the default breakpoint component will show content up to the first breakpoint. The next one will show content up to the next one, and so on.

```.jsx
    <BreakpointsContainer>
         <Breakpoint screenSize="xs">
            <Card background={magma.colors.pop04}>
                <CardBody><strong>Extra-Small: </strong>This will be shown from 0 pixels, up to 959px.</CardBody>
            </Card>
        </Breakpoint>

        <Breakpoint screenSize="medium">
            <Card isInverse background={magma.colors.pop01}>
                <CardBody><strong>Medium: </strong>This text will be visible when the browser width is 960px and larger.</CardBody>
            </Card>
        </Breakpoint>
    </BreakpointsContainer>
```

## Implementation Type

The `implementation` prop can be used to determine whether the browser uses CSS or JavaScript to determine what is shown.  
When `css` is used, the hidden content is simply hidden using CSS `display: none`. When `js` is used, the content to be hidden does not render in the DOM.  
The default implementation is `js`.

```.jsx
    <BreakpointsContainer implementation='css'>
         <Breakpoint screenSize="xs">
            <Card background={magma.colors.pop04}>
                <CardBody><strong>Extra-Small: </strong>This will be shown from 0 pixels, up to 959px.</CardBody>
            </Card>
        </Breakpoint>

        <Breakpoint screenSize="medium">
            <Card isInverse background={magma.colors.pop01}>
                <CardBody><strong>Medium: </strong>This text will be visible when the browser width is 960px and larger.</CardBody>
            </Card>
        </Breakpoint>
    </BreakpointsContainer>
```

## useMediaQuery Hook

The `useMediaQuery` can be used to set a prop based on screen size.

```.jsx
    () => {
      const isSmallerScreen = useMediaQuery(`(max-width:${magma.breakpoints.large}px)`);

      return (
            <Button isFullWidth={isSmallerScreen}>Full Width at Small Screens</Button>
      )}
```

## Custom Breakpoints

Custom breakpoints can be shown by using the `breakpoints` object prop.

```.jsx
    ()=> {

    const customBreakpoints = {
    xs: 0,
    small: 400,
    medium: 760,
    large: 1080
  };

    return(
        <BreakpointsContainer breakpoints={customBreakpoints}>
            <Breakpoint screenSize="xs">
                <Card background={magma.colors.pop04}>
                    <CardBody><strong>Extra-Small: </strong>This will be shown from 0 pixels, up to 499px.</CardBody>
                </Card>
            </Breakpoint>


            <Breakpoint screenSize="small">
                <Card isInverse background={magma.colors.foundation02}>
                    <CardBody><strong>Small: </strong>This will be shown from 400px pixels up to 759px.</CardBody>
                </Card>
            </Breakpoint>

            <Breakpoint screenSize="medium">
                <Card isInverse background={magma.colors.pop01}>
                    <CardBody><strong>Medium: </strong>This text will be visible when the browser width is 760px and up to 1079px.</CardBody>
                </Card>
            </Breakpoint>

            <Breakpoint screenSize="large">
                <Card isInverse background={magma.colors.success01}>
                    <CardBody><strong>Large: </strong>This text will be visible when the browser width is 1080px and greater.</CardBody>
                </Card>
            </Breakpoint>
        </BreakpointsContainer>
    )
}
```
