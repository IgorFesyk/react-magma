# Typography

The `Typography` component is used to present your content in a clear and consistent manner, by limiting the number to sizes and styles with a UI. This component will display variants for different heading and paragraph styles.

The default varaint is `bodyMedium`.

## Basic Usage

```.jsx
    <>
        <Typography variant="headingXLarge">
            Heading XLarge
        </Typography>

        <Typography variant="headingLarge">
            Heading Large
        </Typography>

        <Typography variant="headingMedium">
            Heading Medium
        </Typography>

        <Typography variant="headingSmall">
            Heading Small
        </Typography>

        <Typography variant="headingXSmall">
            Heading XSmall
        </Typography>

        <Typography variant="headingXXSmall">
            Heading XXSmall
        </Typography>

        <Typography variant="bodyLarge">
            Body large.  Lorem ipsum dolar sit amet.
        </Typography>

        <Typography>
            Body medium (default).  Lorem ipsum dolar sit amet.
        </Typography>

        <Typography variant="bodySmall">
            Body small.  Lorem ipsum dolar sit amet.
        </Typography>

        <Typography variant="bodyXSmall">
            Body xsmall.  Lorem ipsum dolar sit amet.
        </Typography>
    </>
```

## Margins

By default, the `Typography` component does not include margins. When the `hasMargins` prop is applied, the elements will receive bottom and top elements for their respective styles.

```.jsx
    <>
        <Typography hasMargins variant="headingXLarge">
            Heading XLarge
        </Typography>

        <Typography hasMargins variant="headingLarge">
            Heading Large
        </Typography>

        <Typography hasMargins variant="headingMedium">
            Heading Medium
        </Typography>

        <Typography hasMargins variant="headingSmall">
            Heading Small
        </Typography>

        <Typography hasMargins variant="headingXSmall">
            Heading XSmall
        </Typography>

        <Typography variant="headingXXSmall">
            Heading XXSmall
        </Typography>

        <Typography hasMargins variant="bodyLarge">
            Body large.  Lorem ipsum dolar sit amet.
        </Typography>

        <Typography hasMargins variant="bodyMedium">
            Body medium.  Lorem ipsum dolar sit amet.
        </Typography>

        <Typography hasMargins variant="bodySmall">
            Body small.  Lorem ipsum dolar sit amet.
        </Typography>

        <Typography hasMargins variant="bodyXSmall">
            Body xsmall.  Lorem ipsum dolar sit amet.
        </Typography>
    </>
```

## Component

By default, each of the variants will render as its corresponding semantic element. For example, `headingXLarge` will render an `h1`, `headingLarge` will render and `h2` and so on. Each of the `body` variants will render with a `p` tag.

However, you can easily override which element is rendered, by using the `component` prop.

```.jsx
    <>
        <Typography hasMargins variant="headingXLarge" component="h2">
            Heading XLarge as H2
        </Typography>

        <Typography hasMargins variant="bodyLarge" component="span">
            Body large as span.  Lorem ipsum dolar sit amet.
        </Typography>
    </>
```

## Type-Style

The `expressive` type-style can be used in marketing contexts, which may require more variety than the default productive styles.

```.jsx
    <>
        <Typography hasMargins typeStyle="expressive" variant="headingXLarge">
            Expressive Heading XLarge
        </Typography>

        <Typography hasMargins typeStyle="expressive" variant="headingLarge">
            Expressive Heading Large
        </Typography>

        <Typography hasMargins typeStyle="expressive" variant="headingMedium">
            Expressive Heading Medium
        </Typography>

        <Typography hasMargins typeStyle="expressive" variant="headingSmall">
            Expressive Heading Small
        </Typography>

        <Typography hasMargins typeStyle="expressive" variant="headingXSmall">
            Expressive Heading XSmall
        </Typography>

        <Typography hasMargins typeStyle="expressive" variant="headingXXSmall">
            Expressive Heading XXSmall
        </Typography>

        <Typography hasMargins typeStyle="expressive" variant="bodyLarge">
            Expressive body large.  Lorem ipsum dolar sit amet.
        </Typography>
    </>
```

## Color

The `color` prop can be used to change the style or emphasis on the different variants. It accepts the values: `success`, `danger` and `subdued`.

```.jsx
    <>
        <Typography color="success" hasMargins variant="headingMedium">
            Positive Heading
        </Typography>

        <Typography color="subdued" hasMargins variant="bodyMedium">
            Subdued text.  Lorem ipsum dolar sit amet.
        </Typography>

        <Typography hasMargins variant="bodyMedium">
            Some <Typography component="span" color="danger" variant="bodyMedium"> inline negative text</Typography>. And some <strong>bold</strong> and <em>italic</em> text. Lorem ipsum dolar sit amet.
        </Typography>

         <Typography color="success" hasMargins variant="headingMedium">
            Expressive Positive Heading
        </Typography>

        <Typography color="subdued" hasMargins variant="bodyLarge">
            Expressive subdued text.  Lorem ipsum dolar sit amet.
        </Typography>

        <Typography hasMargins variant="bodyLarge">
            Some expressive <Typography component="span" color="danger" variant="bodyLarge"> inline negative text</Typography>. And some <strong>bold</strong> and <em>italic</em> text. Lorem ipsum dolar sit amet.
        </Typography>
    </>
```

## Inverse

The `isInverse` property is an optional boolean, that changes the colors so component can appear on a dark background.

```.jsx
    <div style={{background: '#003865', padding: '4px 16px'}}>
        <Typography color="success" hasMargins isInverse variant="headingMedium">
            Normal Heading
        </Typography>

        <Typography hasMargins isInverse variant="bodyMedium">
            Normal text.  Lorem ipsum dolar sit amet.
        </Typography>

         <Typography color="subdued" hasMargins isInverse variant="headingMedium">
            Subdued Heading
        </Typography>

        <Typography color="subdued" hasMargins isInverse variant="bodyMedium">
            Subdued text.  Lorem ipsum dolar sit amet.
        </Typography>
    </div>
```
