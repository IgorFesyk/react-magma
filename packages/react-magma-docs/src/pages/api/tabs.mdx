# Tabs

## Basic Usage

The tab system is made up of four components:
`TabContainer`, which contains both the tabs navigation and the respective content. `Tabs`, which contains the tabs navigation bar.
`Tab`, which is each individual menu item within the tabs navigation bar. And `TabPanel`, which contains the indivdual content sections for each tab.

```.jsx
      function TabComponent() {

        return (
          <TabsContainer activeIndex={1}>
            <Tabs backgroundColor='#F7F7F7'>
              <Tab>Main page with a longer text label</Tab>
              <Tab>FAQ</Tab>
              <Tab>About us</Tab>
            </Tabs>

            <TabPanel index={0}>
              <div>Main page</div>
            </TabPanel>
            <TabPanel index={1}>
              <div>FAQ</div>
            </TabPanel>
            <TabPanel index={2}>
              <div>About us</div>
            </TabPanel>
          </TabsContainer>
    )}
```

# Border Position

By default, horizontal tabs have the inidicator placed underneath the tabs.
However, there are times you may want to place the indicator above the tabs, such as when the tab menu is displayed below its content.
The `borderPosition` prop is an enum and accepts `top` or `bottom` for horizontal tabs. The border position for vertical tabs is always `left`.

The `borderPosition` prop is specified on the `Tabs` component and will apply to each individual tab.

```.jsx
      function TabComponent() {

        return (
          <div>
            <TabsContainer>
              <TabPanel index={0}>
                  <div>Main page <br/><br/></div>
                </TabPanel>
                <TabPanel index={1}>
                  <div>FAQ <br/><br/></div>
                </TabPanel>
                <TabPanel index={2}>
                  <div>About us <br/><br/></div>
                </TabPanel>
                <Tabs backgroundColor='#F7F7F7' borderPosition='top'>
                  <Tab>Main page</Tab>
                  <Tab>FAQ</Tab>
                  <Tab>About us</Tab>
                </Tabs>
            </TabsContainer>
          </div>
    )}
```

## Custom Logic

This is a simple example of Tab Components with custom switching logic

```.jsx
      class TabComponent extends React.Component {
        constructor(props) {
          super(props);
          this.state = {
            activeIndex: 0,
          }
          this.handleChange = this.handleChange.bind(this)
        }

        handleChange(index) {
          this.setState(() => ({
            activeIndex: index,
          }))
        }

        render() {
          return (
            <TabsContainer activeIndex={this.state.activeIndex}>
                <Tabs
                  backgroundColor='#F7F7F7'
                  onChange={this.handleChange}
                >
                  <Tab index={0}>First item</Tab>
                  <Tab index={1}>Second item</Tab>
                  <Tab index={2}>Third item</Tab>
                </Tabs>

                <TabPanel index={0}>
                  <div>First item</div>
                </TabPanel>
                <TabPanel index={1}>
                  <div>Second item</div>
                </TabPanel>
                <TabPanel index={2}>
                  <div>Third item</div>
                </TabPanel>
            </TabsContainer>
          )
        }
      }

```

## Disabled Tabs

If a `Tab` is `disabled`, it will not be actionable. It will still appear in the menu, but will be styled to appear greyed out.

```.jsx
    <TabsContainer>
        <Tabs backgroundColor='#F7F7F7'>
          <Tab>Main page</Tab>
          <Tab disabled>FAQ</Tab>
          <Tab>About us</Tab>
        </Tabs>

        <TabPanel index={0}>
          <div>Main page</div>
        </TabPanel>
        <TabPanel index={1}>
          <div>FAQ</div>
        </TabPanel>
        <TabPanel index={2}>
          <div>About us</div>
        </TabPanel>
    </TabsContainer>

```

## Centered Tabs

The `isCentered` prop is an optional boolean that will style the tabs in the center of their container.

```.jsx
      function TabComponent() {
        return (
          <TabsContainer>
              <Tabs isCentered>
                <Tab>Main page</Tab>
                <Tab>FAQ</Tab>
                <Tab>About us</Tab>
              </Tabs>
          </TabsContainer>
    )}
```

## Full-Width Tabs

The `isFullWidth` prop is an optional boolean that will cause the tabs to take up the full width of their container. The `isFullWidth` prop is specified on the `Tabs` component.

```.jsx
      function TabComponent() {
        return (
          <TabsContainer>
            <Tabs isFullWidth>
              <Tab>Main Page with a longer text label</Tab>
              <Tab>FAQ</Tab>
              <Tab>About Us</Tab>
            </Tabs>
        </TabsContainer>
    )}
```

## Scrollable Tabs with Buttons

## Vertical Tabs

The `orientation` prop can be used to display tabs vertically, instead of horizontalally.  
The `orientation` prop is an enum that accepts either `horizontal` or `vertical` with `horizontal` being the default.
This prop is specified on the `Tabs` component.

```.jsx
      function TabComponent() {
        return (
          <div>
            <TabsContainer>
                <Tabs orientation='vertical' backgroundColor='#F7F7F7'>
                  <Tab>Main page with a longer text label</Tab>
                  <Tab>FAQ</Tab>
                  <Tab>About us</Tab>
                </Tabs>

                <div style={{padding: '10px'}}>
                <TabPanel index={0}>
                  <div>Main page</div>
                </TabPanel>
                <TabPanel index={1}>
                  <div>FAQ</div>
                </TabPanel>
                <TabPanel index={2}>
                  <div>About us</div>
                </TabPanel>
                </div>
            </TabsContainer>
          </div>
    )}
```

## Nav Tabs

This is an example with custom element in Tabs.
By default we use `button` element in Tabs, but you can include your custom element inside a `Tab` component.
In this case, please put all children inside the `component` prop instead of putting the content directly inside each `Tab`.
If you would like to use `Tab` like an route switcher, keep the `path` prop in `Tab` equal to desired path.

```.jsx
    <Tabs>
      <Tab path="/main" component={<Link to="#">Main page</Link>}/>
      <Tab path="/faq" component={<Link to="#">FAQ</Link>}/>
      <Tab path="/about" component={<Link to="#">About us</Link>}/>
    </Tabs>
```

## Icon Tabs

The `icon` prop can be used to display an icon inside the individual `Tab` components.

### Icon Only Tabs

`Tab` components without children can be used to display an icon only version of the tabs. Be sure to include an `ariaLabel` for each tab, when using this pattern.

```.jsx
      function TabComponent() {
        return (
          <TabsContainer>
              <Tabs>
                <Tab ariaLabel='Graduation' icon={<GraduationCapIcon />} />
                <Tab ariaLabel='Android' icon={<AndroidIcon />} />
                <Tab ariaLabel='Bell' icon={<BellIcon />} />
              </Tabs>
          </TabsContainer>
    )}
```

### Icon With Text

`Tab` components with children and and `icon` prop will display both the icon and the children.
By default, the icons appear above the children.

```.jsx
      function TabComponent() {
        return (
          <TabsContainer>
              <Tabs>
                <Tab icon={<GraduationCapIcon />}>First item with a longer text label</Tab>
                <Tab icon={<AndroidIcon />}>Second item</Tab>
                <Tab icon={<BellIcon />}>Third item</Tab>
              </Tabs>
          </TabsContainer>
    )}
```

### Icon With Text, Icon Left-Aligned

The `iconPosition` prop can be used to position the icon to the left of the children.  
The `iconPosition` prop is an enum that accepts either `left` or `top` with `top` being the default for horizontal tabs.
This prop is specified on the `Tabs` component.

```.jsx
    <TabsContainer>
        <Tabs iconPosition="left" backgroundColor='#F7F7F7'>
          <Tab icon={<GraduationCapIcon />}>First item with a longer text label</Tab>
          <Tab icon={<AndroidIcon />}>Second item</Tab>
          <Tab icon={<BellIcon />}>Third item</Tab>
        </Tabs>
    </TabsContainer>
```

## Inverse

The `isInverse` property is an optional boolean, that reverses the colors so that the tabs can better appear on a dark background.
The default value is false.

### With Blue Background

```.jsx
      function TabComponent() {
        const wrapperStyles = {
          backgroundColor: '#0f3765'
        }

        return (
          <TabsContainer style={wrapperStyles}>
              <Tabs isInverse={true}>
                <Tab>First item</Tab>
                <Tab>Second item</Tab>
                <Tab>Third item</Tab>
              </Tabs>
          </TabsContainer>
    )}
```

### With Dark Background

```.jsx
function TabComponent() {

  const wrapperStyles = {
    backgroundColor: '#3F3F3F'
  }

  return (
    <TabsContainer style={wrapperStyles}>
        <Tabs isInverse={true}>
          <Tab>First item</Tab>
          <Tab>Second item</Tab>
          <Tab>Third item</Tab>
        </Tabs>
    </TabsContainer>
  )
}
```
