# Paragraphs

The `Paragraph` component is used to present paragraph content in a clear and consistent manner, by limiting the number to sizes and styles with a UI.

See also: the <Link to="/api/span">Span</Link> component and the <Link to="api/heading">Heading</Link> component.

## Basic Usage

```.jsx
    <Paragraph>
        Basic paragraph component.
    </Paragraph>
```

## Sizes

There are four relative sizes for the paragraph: `bodyLarge`, `bodyMedium`, `bodySmall`, and `bodyXSmall`.
The default size is `bodyMedium`.

```.jsx
    <>
        <Paragraph size="bodyLarge">
            Body large.  Lorem ipsum dolar sit amet.
        </Paragraph>

        <Paragraph size="bodyMedium">
            Body medium (default).  Lorem ipsum dolar sit amet.
        </Paragraph>

        <Paragraph size="bodySmall">
            Body small.  Lorem ipsum dolar sit amet.
        </Paragraph>

        <Paragraph size="bodyXSmall">
            Body xsmall.  Lorem ipsum dolar sit amet.
        </Paragraph>
    </>
```

## Heading Sizes

In addition to the paragraph relative sizes, the `sizes` prop can also accept the values for the different heading sizes that are used with the <Link to="/api/heading">Heading</Link> component.
The underlying HTML will still render as a `p` element.

```.jsx
  <>
    <Paragraph size="headingXLarge">
        Heading large.  Lorem ipsum dolar sit amet.
    </Paragraph>

    <Paragraph size="headingLarge">
        Heading large.  Lorem ipsum dolar sit amet.
    </Paragraph>

    <Paragraph size="headingMedium">
        Heading large.  Lorem ipsum dolar sit amet.
    </Paragraph>

    <Paragraph size="headingSmall">
        Heading large.  Lorem ipsum dolar sit amet.
    </Paragraph>

    <Paragraph size="headingXSmall">
        Heading large.  Lorem ipsum dolar sit amet.
    </Paragraph>

    <Paragraph size="headingXXSmall">
        Heading large.  Lorem ipsum dolar sit amet.
    </Paragraph>
  </>
```

## Variant

The `variant` prop can be used to set additional styles on a component. The default `variant` is `default`.

The `expressive` variant can be used in marketing contexts, which may require more variety than the default styles.

The `narrative` variant can be used for content-heavy contexts, such as an e-reader.

```.jsx
    <>

        <Paragraph variant="expressive" size="bodyLarge">
            Body large.  Lorem ipsum dolar sit amet.
        </Paragraph>

        <Paragraph variant="expressive" size="bodyMedium">
            Body medium (default).  Lorem ipsum dolar sit amet.
        </Paragraph>

        <Paragraph variant="expressive" size="bodySmall">
            Body small.  Lorem ipsum dolar sit amet.
        </Paragraph>

        <Paragraph variant="expressive" size="bodyXSmall">
            Body xsmall.  Lorem ipsum dolar sit amet.
        </Paragraph>

        <Paragraph variant="narrative" size="bodyLarge">
            Body large.  Lorem ipsum dolar sit amet.
        </Paragraph>

        <Paragraph variant="narrative" size="bodyMedium">
            Body medium (default).  Lorem ipsum dolar sit amet.
        </Paragraph>

        <Paragraph variant="narrative" size="bodySmall">
            Body small.  Lorem ipsum dolar sit amet.
        </Paragraph>

        <Paragraph variant="narrative" size="bodyXSmall">
            Body xsmall.  Lorem ipsum dolar sit amet.
        </Paragraph>
    </>
```

## Color

The `color` prop can be used to change the style or emphasis on the different variants.
It accepts the values: `success`, `danger` and `subdued`.

There are no inverse styles for the `success` or `danger` colors.

```.jsx
    <>
        <Paragraph color="success" size="bodyLarge">
            Success paragraph. Lorem ipsum dolar sit amet.
        </Paragraph>

        <Paragraph color="subdued" size="bodyMedium">
            Subdued paragraph.  Lorem ipsum dolar sit amet.
        </Paragraph>

        <Paragraph color="danger" v="bodySmall">
            Danger paragraph.  Lorem ipsum dolar sit amet.
        </Paragraph>
    </>
```

## Inverse

The `isInverse` property is an optional boolean that changes the colors so the component can appear on a dark background.

There are no inverse styles for the `success` or `danger` colors.

```.jsx
    <div style={{background: '#003865', padding: '4px 16px'}}>
         <Paragraph color="subdued" isInverse size="bodyMedium">
            Inverse paragraph.  Lorem ipsum dolar sit amet.
        </Paragraph>
        <Paragraph isInverse size="bodyMedium">
            Subdued inverse paragraph.  Lorem ipsum dolar sit amet.
        </Paragraph>
    </div>
```

## No Margins

By default, the `Paragraph` component includes margins on the top and bottom. When the `noMargins` prop is applied, the element will have a margin value of 0.

```.jsx
  <>
     <Paragraph noMargins size="bodyLarge">
            Body large.  Lorem ipsum dolar sit amet.
        </Paragraph>

        <Paragraph noMargins size="bodyMedium">
            Body medium (default).  Lorem ipsum dolar sit amet.
        </Paragraph>

        <Paragraph noMargins size="bodySmall">
            Body small.  Lorem ipsum dolar sit amet.
        </Paragraph>

        <Paragraph noMargins size="bodyXSmall">
            Body xsmall.  Lorem ipsum dolar sit amet.
        </Paragraph>
  </>
```
