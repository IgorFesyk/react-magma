# Paragraphs

The `Paragraph` component is used to present paragraph content in a clear and consistent manner, by limiting the number to sizes and styles with a UI.

## Basic Usage

```.jsx
    <Paragraph>
        Basic paragraph component.
    </Paragraph>
```

## Sizes

There are four visual styles for the paragraph: `bodyLarge`, `bodyMedium`, `bodySmall`, and `bodyXSmall`, which determine the paragraph's relative font-size.
The default `visualStyle` is `bodyMedium`.

In addition to the paragraph visual styles, the `visualStyle` prop can also accept the values for the different heading visual styles: `headingXLarge`, `headingLarge`, `headingMedium`, `headingSmall`, `headingXSmall` and `heading2XSmall`.
These visual styles are also used with the <Link to="/api/heading">Heading</Link> component. The underlying HTML will still render as a `p` element.

```.jsx
  <>
    <Paragraph visualStyle="bodyLarge">
        Body large.  Lorem ipsum dolar sit amet.
    </Paragraph>

    <Paragraph visualStyle="bodyMedium">
        Body medium (default).  Lorem ipsum dolar sit amet.
    </Paragraph>

    <Paragraph visualStyle="bodySmall">
        Body small.  Lorem ipsum dolar sit amet.
    </Paragraph>

    <Paragraph visualStyle="bodyXSmall">
        Body x-small.  Lorem ipsum dolar sit amet.
    </Paragraph>

    <Paragraph visualStyle="headingXLarge">
        Heading large.  Lorem ipsum dolar sit amet.
    </Paragraph>

    <Paragraph visualStyle="headingLarge">
        Heading large.  Lorem ipsum dolar sit amet.
    </Paragraph>

    <Paragraph visualStyle="headingMedium">
        Heading large.  Lorem ipsum dolar sit amet.
    </Paragraph>

    <Paragraph visualStyle="headingSmall">
        Heading large.  Lorem ipsum dolar sit amet.
    </Paragraph>

    <Paragraph visualStyle="headingXSmall">
        Heading large.  Lorem ipsum dolar sit amet.
    </Paragraph>

    <Paragraph visualStyle="heading2XSmall">
        Heading large.  Lorem ipsum dolar sit amet.
    </Paragraph>
  </>
```

## Variant

The `variant` prop can be used to set additional styles on a component. The default `variant` is `default`.

The `expressive` variant can be used in marketing contexts, which may require more variety than the default styles.

The `narrative` variant can be used for content-heavy contexts, such as a blog-post or an e-reader.

```.jsx
    <>

        <Paragraph contextVariant="expressive" visualStyle="bodyLarge">
            Body large.  Lorem ipsum dolar sit amet.
        </Paragraph>

        <Paragraph contextVariant="expressive" visualStyle="bodyMedium">
            Body medium (default).  Lorem ipsum dolar sit amet.
        </Paragraph>

        <Paragraph contextVariant="expressive" visualStyle="bodySmall">
            Body small.  Lorem ipsum dolar sit amet.
        </Paragraph>

        <Paragraph contextVariant="expressive" visualStyle="bodyXSmall">
            Body xsmall.  Lorem ipsum dolar sit amet.
        </Paragraph>

        <Paragraph contextVariant="narrative" visualStyle="bodyLarge">
            Body large.  Lorem ipsum dolar sit amet.
        </Paragraph>

        <Paragraph contextVariant="narrative" visualStyle="bodyMedium">
            Body medium (default).  Lorem ipsum dolar sit amet.
        </Paragraph>

        <Paragraph contextVariant="narrative" visualStyle="bodySmall">
            Body small.  Lorem ipsum dolar sit amet.
        </Paragraph>

        <Paragraph contextVariant="narrative" visualStyle="bodyXSmall">
            Body xsmall.  Lorem ipsum dolar sit amet.
        </Paragraph>
    </>
```

## Color

The `color` prop can be used to change the color of the paragraph to convey additional meaning. It accepts the values: `success`, `danger` and `subdued`.

```.jsx
    <>
        <Paragraph color="success">
            Success paragraph. Lorem ipsum dolar sit amet.
        </Paragraph>

        <Paragraph color="subdued">
            Subdued paragraph.  Lorem ipsum dolar sit amet.
        </Paragraph>

        <Paragraph color="danger">
            Danger paragraph.  Lorem ipsum dolar sit amet.
        </Paragraph>
    </>
```

## Inverse

The `isInverse` property is an optional boolean that changes the colors so the component can appear on a dark background.

```.jsx
    <div style={{background: '#003865', padding: '4px 16px'}}>
         <Paragraph isInverse>
            Inverse paragraph.  Lorem ipsum dolar sit amet.
        </Paragraph>

         <Paragraph color="success" isInverse>
            Success inverse paragraph. Lorem ipsum dolar sit amet.
        </Paragraph>

        <Paragraph color="subdued" isInverse>
            Subdued inverse paragraph.  Lorem ipsum dolar sit amet.
        </Paragraph>

        <Paragraph color="danger" isInverse>
            Danger inverse paragraph.  Lorem ipsum dolar sit amet.
        </Paragraph>
    </div>
```

## No Margins

By default, the `Paragraph` component includes margins on the top and bottom. When the `noMargins` prop is applied, the element will have a margin value of 0.

```.jsx
  <>
     <Paragraph noMargins visualStyle="bodyLarge">
            Body large.  Lorem ipsum dolar sit amet.
        </Paragraph>

        <Paragraph noMargins visualStyle="bodyMedium">
            Body medium (default).  Lorem ipsum dolar sit amet.
        </Paragraph>

        <Paragraph noMargins visualStyle="bodySmall">
            Body small.  Lorem ipsum dolar sit amet.
        </Paragraph>

        <Paragraph noMargins visualStyle="bodyXSmall">
            Body x-small.  Lorem ipsum dolar sit amet.
        </Paragraph>
  </>
```
