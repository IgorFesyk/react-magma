# Icon Buttons

**Icon Buttons use the <Link to="/api/button">Buttons component</Link> to render. It is not a separate component, just different props.**

## Basic Usage

Icon buttons have an icon included in them. If children are included with these buttons (such as text), then they render in a style similar to the non-icon buttons.
If no children are included, then they render in the icon-only style.

```.jsx
    <>
        <p>
          <Button aria-label="Icon" icon={<BellIcon />} />
          <Button aria-label="Icon" icon={<BellIcon />} />
          <Button aria-label="Icon" variant="outline" icon={<BellIcon />} />
          <Button aria-label="Icon" variant="link" icon={<BellIcon />} />
          <Button aria-label="Icon" color="secondary" icon={<Cog2Icon />} />
          <Button aria-label="Icon" color="secondary" variant="outline" icon={<Cog2Icon />} />
          <Button aria-label="Icon" color="secondary" variant="link" icon={<Cog2Icon />} />
          <Button aria-label="Icon" color="success" icon={<CheckIcon />} />
          <Button aria-label="Icon" color="success"variant="outline" icon={<CheckIcon />} />
          <Button aria-label="Icon" color="success" variant="link" icon={<CheckIcon />} />
          <Button aria-label="Icon" color="danger" icon={<AlertIcon />} />
          <Button aria-label="Icon" color="danger" variant="outline" icon={<AlertIcon />} />
          <Button aria-label="Icon" color="danger" variant="link" icon={<AlertIcon />} />
          <Button aria-label="Icon" disabled icon={<BellIcon />} />
        </p>
         <p>
          <Button icon={<BellIcon />}>Button with text</Button>
          <Button icon={<BellIcon />} color="secondary" variant="outline" >Button with text</Button>
          <Button icon={<BellIcon />} color="success" variant="link" >Button with text</Button>
        </p>
    </>
```

## Assign Icon to Variable

If you find JSX being passed in as a prop directly ugly, you can assign an icon to a variable and pass that in as the prop.

```.jsx
  class Example extends React.Component {
      render() {
        const icon = <BellIcon />;

        return <Button aria-label="Icon" icon={icon} />
      }
    }
```

For icon only buttons you _MUST_ pass in an `aria-label` property to satisfy accessibility standards.

## Aria-Label

```.jsx
    <>
        <Button aria-label="Icon" icon={<BellIcon />} />
    </>
```

## Icon Position

Use the optional prop `iconPosition` to position the icon of the button on either side of the text.
The `iconPosition` property takes the values `left` and `right`.

```.jsx
    <>
        <Button icon={<BellIcon />}>Icon Left</Button>
        <Button color="secondary" icon={<InfoIcon />} iconPosition="right">Icon Right</Button>
    </>
```

## Sizes

Sizes for `IconButtons` include `small`, `medium`, and `large`, with `medium` being the default value.

```.jsx
    <>
      <p>
          <Button icon={<BellIcon />} size="small" />
          <Button icon={<BellIcon />} />
          <Button icon={<BellIcon />} size="large" />
      </p>
      <p>
          <Button icon={<BellIcon />} size="small" iconPosition="left">Small</Button>
          <Button icon={<BellIcon />} iconPosition="left">Medium</Button>
          <Button icon={<BellIcon />} size="large" iconPosition="left">Large</Button>
      </p>
    </>
```

## Shape

Icon only buttons have `shape = round`, by default.

```.jsx
    <>
      <Button icon={<BellIcon />} />
      <Button icon={<BellIcon />} shape="fill" />
      <Button icon={<BellIcon />} shape="rightCap" />
      <Button icon={<BellIcon />} shape="leftCap" />
    </>
```

## Block Buttons

Passing the `block` property will set the button to display full-width.
Block buttons only work when the `icon` property is set to `false` or not defined.

```.jsx
    <>
        <Button icon={<BellIcon />} block iconPosition="left">Block button</Button>
        <Button icon={<BellIcon />} color="secondary" block iconPosition="right">Block button</Button>
    </>
```

## Event Handling

The `onClick` event will fire when the button is clicked, or when the enter or space bar key is pressed.
Events do not fire on disabled buttons.

```.jsx
  class Example extends React.Component {
      constructor(props) {
        super(props)
        this.state = {
          iconButtonCounter: 0,
          textIconButtonCounter: 0
        }
      }

      render() {
        return <>
          <p><strong>Icon Counter: </strong> {this.state.iconButtonCounter} | <strong>Text Icon Counter: </strong> {this.state.textIconButtonCounter}</p>
          <Button
            icon={<BellIcon />}
            onClick={() => {
              this.setState(state => ({iconButtonCounter: state.iconButtonCounter + 1}))
            }}>Click me</Button>
          <Button
            icon={<BellIcon />}
            textPosition="left"
            onClick={() => {
              this.setState(state => ({textIconButtonCounter: state.textIconButtonCounter + 1}))
            }}>Click me</Button>
          <Button
            disabled
            icon={<BellIcon />}
            onClick={() => {
              this.setState(state => ({iconButtonCounter: state.iconButtonCounter + 1}))
            }}>Click me if you can</Button>
        </>
      }
    }
```

## Testing

Passing in the `testId` prop will add the `data-testid` attribute to the button element for easier querying in tests.

```html

    class="icon"
    height="18"
    width="18"
    fill="currentColor"
    xmlns="http://www.w3.org/2000/svg"
    viewBox="0 0 21.17 23.56"
    aria-labelledby="5c3f5f70-81b2-4793-90ad-f69fca370f6a"
  >
    <title id="5c3f5f70-81b2-4793-90ad-f69fca370f6a"></title>
    <path
      d="M21.9,10.31c0-4.8-5.34-6.49-5.34-6.49V2.22h-2v1.6S9.22,5.51,9.22,10.31,7,20.69,5,20.69v1.9h7.38a3.2,3.2,0,0,0,6.39,0h7.39v-1.9c-2,0-4.24-5.59-4.24-10.38Z"

  </svg>
</button>
```
