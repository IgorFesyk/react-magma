## Basic usage

Buttons are clickable items used to perform an action. See the <Link to="/design/button/">Button Design Guidelines</Link> for more information on when to use the different kinds of buttons.

When used with icons, the Button component takes a slightly different set of props. For information on how to use this component with icons, please see documentation on <Link to="/api/icon-button">Icon Buttons</Link>.

## Sizes

Sizes for buttons include `small`, `medium`, and `large`, with `medium` being the default value.

```.jsx
    <div>
        <Button size="small">Small</Button>
        <Button>Default</Button>
        <Button size="large">Large</Button>
    </div>
```

## Colors

Colors for buttons include `primary`, `secondary`, `success`, `danger`, and `marketing` with `primary` being the default value.

```.jsx
    <div>
        <Button color="primary">Primary (default)</Button>
        <Button color="secondary">Secondary</Button>
        <Button color="success" >Success</Button>
        <Button color="danger">Danger</Button>
        <Button color="marketing">Marketing</Button>
    </div>
```

## Variants

Variants for buttons include `solid`, `outline`, and `link`, with `solid` being the default value.

The `marketing` buttons do not have `outline` and `link` variants, and will render as solid buttons.

```.jsx
    <div>
      <p>
          <Button color="primary" variant="solid">Solid (default)</Button>
          <Button color="primary" variant="outline">Outline</Button>
          <Button color="primary" variant="link">Link</Button>
      </p>
      <p>
          <Button color="secondary" variant="solid">Solid (default)</Button>
          <Button color="secondary" variant="outline">Outline</Button>
          <Button color="secondary" variant="link">Link</Button>
      </p>
      <p>
          <Button color="success" variant="solid">Solid (default)</Button>
          <Button color="success" variant="outline">Outline</Button>
          <Button color="success" variant="link">Link</Button>
      </p>
      <p>
          <Button color="danger" variant="solid">Solid (default)</Button>
          <Button color="danger" variant="outline">Outline</Button>
          <Button color="danger" variant="link">Link</Button>
      </p>
    </div>
```

## Link Buttons

Using the `LinkButton` will render a button that is styled exactly like a link. Unlike the using `<Button variant='link' />`,
the `LinkButton` will render as inline text, the way an `a` element would.

```.jsx
    <div>
        This is a <LinkButton>link button</LinkButton>.
    </div>
```

## Shapes

Shapes for buttons include `fill`, `leftCap`, and `rightCap`, with `fill` being the default value.

```.jsx
    <div>
        <Button color="primary">Fill (default)</Button>
        <Button color="primary" shape="leftCap">Left Cap</Button>
        <Button color="primary" shape="rightCap">Right Cap</Button>
    </div>
```

## Inverse

The `inverse` property is an optional boolean, that reverses the colors so that the buttons can better appear on a dark background.
The default value is `false`.

While `outline inverse buttons` and `link inverse buttons` still accept the `color` property, the `color` property
online affects `solid inverse buttons`.

The `marketing` button does not have an inverse state, as it already is styled to appear well against a dark background.

```.jsx
    <div style={{ background : '#00263E', color: '#ffffff', padding: '10px' }}>
      <p>
          <Button inverse>Solid inverse</Button>
          <Button color="secondary" inverse>Solid inverse</Button>
          <Button color="success" inverse>Solid inverse</Button>
          <Button color="danger" inverse>Solid inverse</Button>
          <Button color="marketing" inverse>Marketing inverse</Button>
      </p>
      <p>
          <Button variant="outline" inverse>Outline inverse</Button>
          <Button variant="link" inverse>Link inverse</Button>
      </p>
      <p>
          <Button disabled inverse>Disabled inverse</Button>
          <Button disabled inverse variant="outline">Disabled outline inverse</Button>
          <Button disabled inverse variant="link">Disabled link inverse</Button>
      </p>
      <p>
        This is a <LinkButton inverse>link button</LinkButton>.
      </p>
      <p>
        This is a <LinkButton disabled inverse>disabled link button</LinkButton>.
      </p>
    </div>
```

## Text Transform

Options for the `textTransform` prop for buttons include `uppercase` and `none`, with `uppercase` (all caps) being the default value.
This sets the CSS `text-transform` property.

```.jsx
    <div>
        <Button>All caps</Button>
        <Button textTransform="none">No text tranform</Button>
        <Button textTransform="uppercase">Uppercase tranform</Button>
    </div>
```

## Block Buttons

Passing the `block` property will set the button to display full-width.

```.jsx
    <div>
        <Button block>Block button</Button>
        <Button block color="secondary">Block button</Button>
    </div>
```

## Disabled

```.jsx
    <div>
        <Button disabled>Disabled</Button>
        <Button disabled variant="outline">Disabled</Button>
        <Button disabled variant="link">Disabled</Button>
        <p>This is a <LinkButton disabled>disabled link button</LinkButton>.</p>
    </div>
```

## Types

```.jsx
  class Example extends React.Component {
      constructor(props) {
        super(props);
        this.onFormSubmit = this.onFormSubmit.bind(this);
        this.onFormReset = this.onFormReset.bind(this);

        this.inputRef = React.createRef();
      }

      onFormSubmit(event) {
        event.preventDefault();
        alert('Form Submitted');
      }

      onFormReset(event) {
        event.preventDefault();
        this.inputRef.current.value = '';
      }

      render() {
        return <form onSubmit={this.onFormSubmit} onReset={this.onFormReset}>
          <Input
            ref={this.inputRef}
            labelText='Form Field'
            value='' />
          <Button type="submit">Submit</Button>
          <Button type="reset">Reset</Button>
        </form>
      }
    }
```

## Event Handling

The `onClick` event will fire when the button is clicked, or when the enter or space bar key is pressed.
Events do not fire on disabled buttons.

```.jsx
  class Example extends React.Component {
      constructor(props) {
        super(props)
        this.state = {
          counter: 0
        }
      }

      render() {
        return <div>
        <p><strong>Counter: </strong> <span id="counter">{this.state.counter}</span></p>
        <Button
          onClick={() => {
            this.setState(state => ({counter: state.counter + 1}))
          }}>Click me</Button>

        <Button
          disabled
          onClick={() => {
            this.setState(state => ({counter: state.counter + 1}))
          }}>Click me</Button>
        </div>
      }
    }
```

## Forward Ref

Using React's `forwardRef` feature you can gain access to the reference of the internal button.

```.jsx
  class Example extends React.Component {
      constructor(props) {
        super(props)
        this.buttonRef = React.createRef();
      }

      render() {
        return <div>
        <Button
          ref={this.buttonRef}>Button to be focused</Button>

        <Button
          onClick={() => {
            this.buttonRef.current.focus();
          }}>Click to focus other button</Button>
        </div>
      }
    }
```

## Testing

Passing in the `testId` prop will add the `data-testid` attribute to the button element for easier querying in tests.

```html
<button data-testid="test-id">Default</button>
```
