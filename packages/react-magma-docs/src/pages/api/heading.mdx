# Headings

## Basic Usage

The `Heading` component can be used with the `level` prop to render the HTML heading elements, levels 1-6.

```.jsx
  <>
    <Heading level={1}>Heading 1</Heading>
    <Heading level={2}>Heading 2</Heading>
    <Heading level={3}>Heading 3</Heading>
    <Heading level={4}>Heading 4</Heading>
    <Heading level={5}>Heading 5</Heading>
    <Heading level={6}>Heading 6</Heading>
  </>
```

## Variant

By default, the a heading will render the semantic HTML element with its corresponding styles, as defined in the <Link to="/api/typography">Typography</Link> component.
However, if the `variant` prop is specified, the heading will render with the styles for that variant.

```.jsx
  <>
    <Heading level={2} variant="headingXXSmall">H2 styled as Heading XXSmall (h6)</Heading>
    <Heading level={3} variant="bodyMedium">H3 styled as standard paragraph text</Heading>
  </>
```

## Margins

By default, the `Heading` component does not include margins. When the `hasMargins` prop is applied, the elements will receive bottom and top elements for their respective styles.

```.jsx
  <>
    <Heading hasMargins level={1}>Heading 1</Heading>
    <Heading hasMargins level={2}>Heading 2</Heading>
    <Heading hasMargins level={3}>Heading 3</Heading>
    <Heading hasMargins level={4}>Heading 4</Heading>
    <Heading hasMargins level={5}>Heading 5</Heading>
    <Heading hasMargins level={6}>Heading 6</Heading>
  </>
```

## Type-Style

The `typeStyle` prop can be used to set additional styles on a component. The default `typeStyle` is `productive`.

The `expressive` type-style can be used in marketing contexts, which may require more variety than the default productive styles.

The `narrative` type-style can be used for content-heavy contexts, such as an e-reader.

```.jsx
    <>
      <Heading typeStyle="expressive" level={1}>Expressive Heading 1</Heading>
      <Heading typeStyle="expressive" level={2}>Expressive Heading 2</Heading>
      <Heading typeStyle="expressive" level={3}>Expressive Heading 3</Heading>
      <Heading typeStyle="expressive" level={4}>Expressive Heading 4</Heading>
      <Heading typeStyle="expressive" level={5}>Expressive Heading 5</Heading>
      <Heading typeStyle="expressive" level={6}>Expressive Heading 6</Heading>
      <br/><br/>
      <Heading typeStyle="narrative" level={1}>Narrative Heading 1</Heading>
      <Heading typeStyle="narrative" level={2}>Narrative Heading 2</Heading>
      <Heading typeStyle="narrative" level={3}>Narrative Heading 3</Heading>
      <Heading typeStyle="narrative" level={4}>Narrative Heading 4</Heading>
      <Heading typeStyle="narrative" level={5}>Narrative Heading 5</Heading>
      <Heading typeStyle="narrative"  level={6}>Narrative Heading 6</Heading>
    </>
```

## Color

The `color` prop can be used to change the style or emphasis. It accepts the values: `success`, `danger` and `subdued`.

```.jsx
    <>
        <Heading color="success" level={2}>
            Positive Heading
        </Heading>

        <Heading color="danger" level={2}>
            Danger Heading
        </Heading>

        <Heading color="subdued" level={2}>
            Subdued Heading
        </Heading>

    </>
```

## Inverse

`isInverse` is an optional boolean prop, that may be passed in when the component is to be displayed on a dark background.

```.jsx
  <div style={{ background : '#00263E', padding: '10px' }}>
    <Heading isInverse level={2}>Heading 2</Heading>
    <Heading isInverse level={2} color="subdued">Heading 2 Subdued</Heading>
  </div>
```

## Forward Ref

Using React's `forwardRef` feature you can gain access to the reference of the internal heading.

```.jsx
  (props) => {
    const headingRef = React.useRef();

    return <div>
      <Heading
        ref={headingRef}
        level={1}>
        Heading to Be Focused
      </Heading>

      <Button
        onClick={() => {
          headingRef.current.tabIndex = -1
          headingRef.current.focus();
        }}>Click to focus heading
      </Button>
    </div>
  }
```

## Testing

Passing in the `testId` prop will add the `data-testid` attribute to the heading element for easier querying in tests.

```html
<h1 data-testid="test-id">Heading 1</h1>
```
