# Heading

## Basic Usage

The `Heading` component can be used with the `level` prop to render the HTML heading elements, levels 1-6.

```.jsx
  <>
    <Heading level={1}>Heading 1</Heading>
    <Heading level={2}>Heading 2</Heading>
    <Heading level={3}>Heading 3</Heading>
    <Heading level={4}>Heading 4</Heading>
    <Heading level={5}>Heading 5</Heading>
    <Heading level={6}>Heading 6</Heading>
  </>
```

## Size

There are six visual styles for the heading: `headingXLarge`, `headingLarge`, `headingMedium`, `headingSmall`, `headingXSmall` and `heading2XSmall`. By default each of the visual styles corresponds with the semantic HTML heading elements levels 1-6.  
Visual style and heading level can be set independently so you can specify the correct semantic level and then apply any of the visual styles to change the visual appearance to fit your design.

Furthermore, the `visualStyle` prop can also accept the values for the four different paragraph visual styles: `bodyLarge`, `bodyMedium`, `bodySmall`, and `bodyXSmall`. These are sizes are also used with the <Link to="/api/paragraph">Paragraph</Link> component.

```.jsx
  <>
    <Heading level={2} visualStyle="headingXLarge">H2 styled as Heading XLarge</Heading>
    <Heading level={2} visualStyle="headingLarge">H2 styled as Heading Large</Heading>
    <Heading level={2} visualStyle="headingMedium">H2 styled as Heading Medium</Heading>
    <Heading level={2} visualStyle="headingSmall">H2 styled as Heading Small</Heading>
    <Heading level={2} visualStyle="headingXSmall">H2 styled as Heading XSmall</Heading>
    <Heading level={2} visualStyle="heading2XSmall">H2 styled as Heading 2XSmall</Heading>

    <Heading level={2} visualStyle="bodyLarge">H2 styled as Body Large</Heading>
    <Heading level={2} visualStyle="bodyMedium">H2 styled as Body Medium</Heading>
    <Heading level={2} visualStyle="bodySmall">H2 styled as Body Small</Heading>
    <Heading level={2} visualStyle="bodyXSmall">H2 styled as Body XSmall</Heading>
  </>
```

## Context Variant

The `contextVariant` prop can be used to set additional styles on a heading component. The `contextVariant` can affect the font-size, text-transform, or font-family. The default `contextVariant` is `default`.

The `expressive` context variant can be used in marketing contexts, which may require more variety than the default styles.

The `narrative` context variant can be used for content-heavy contexts, such as a blog post or an e-reader.

```.jsx
    <>
      <Heading contextVariant="expressive" level={1}>Expressive Heading 1</Heading>
      <Heading contextVariant="expressive" level={2}>Expressive Heading 2</Heading>
      <Heading contextVariant="expressive" level={3}>Expressive Heading 3</Heading>
      <Heading contextVariant="expressive" level={4}>Expressive Heading 4</Heading>
      <Heading contextVariant="expressive" level={5}>Expressive Heading 5</Heading>
      <Heading contextVariant="expressive" level={6}>Expressive Heading 6</Heading>
      <br/><br/>
      <Heading contextVariant="narrative" level={1}>Narrative Heading 1</Heading>
      <Heading contextVariant="narrative" level={2}>Narrative Heading 2</Heading>
      <Heading contextVariant="narrative" level={3}>Narrative Heading 3</Heading>
      <Heading contextVariant="narrative" level={4}>Narrative Heading 4</Heading>
      <Heading contextVariant="narrative" level={5}>Narrative Heading 5</Heading>
      <Heading contextVariant="narrative"  level={6}>Narrative Heading 6</Heading>
    </>
```

## Color

The `color` prop can be used to change the color of the heading to convey additional meaning. It accepts the values: `success`, `danger` and `subdued`.

```.jsx
    <>
        <Heading color="success" level={2}>
            Success Color Heading
        </Heading>

        <Heading color="danger" level={2}>
            Danger Color Heading
        </Heading>

        <Heading color="subdued" level={2}>
            Subdued Color Heading
        </Heading>
    </>
```

## Inverse

The `isInverse` property is an optional boolean that changes the colors so the component can appear on a dark background.

```.jsx
  <div style={{ background : '#00263E', padding: '10px' }}>
      <Heading isInverse level={2}>Heading 2</Heading>

      <Heading color="success" isInverse level={2}>
        Success Color Heading
      </Heading>

      <Heading color="danger" isInverse level={2}>
        Danger Color Heading
      </Heading>

      <Heading isInverse level={2} color="subdued">
        Heading 2 Subdued
      </Heading>
  </div>
```

## No Margins

By default, the `Heading` component includes margins on the top and bottom. When the `noMargins` prop is applied, the element will have a margin value of 0.

```.jsx
  <>
    <Heading noMargins level={1}>Heading 1</Heading>
    <Heading noMargins level={2}>Heading 2</Heading>
    <Heading noMargins level={3}>Heading 3</Heading>
    <Heading noMargins level={4}>Heading 4</Heading>
    <Heading noMargins level={5}>Heading 5</Heading>
    <Heading noMargins level={6}>Heading 6</Heading>
  </>
```

## Forward Ref

Using React's `forwardRef` feature you can gain access to the reference of the internal heading.

```.jsx
  (props) => {
    const headingRef = React.useRef();

    return <div>
      <Heading
        ref={headingRef}
        level={1}>
        Heading to Be Focused
      </Heading>

      <Button
        onClick={() => {
          headingRef.current.tabIndex = -1
          headingRef.current.focus();
        }}>Click to focus heading
      </Button>
    </div>
  }
```

## Testing

Passing in the `testId` prop will add the `data-testid` attribute to the heading element for easier querying in tests.

```html
<h1 data-testid="test-id">Heading 1</h1>
```
