# Heading

## Basic Usage

The `Heading` component can be used with the `level` prop to render the HTML heading elements, levels 1-6.

```.jsx
  <>
    <Heading level={1}>Heading 1</Heading>
    <Heading level={2}>Heading 2</Heading>
    <Heading level={3}>Heading 3</Heading>
    <Heading level={4}>Heading 4</Heading>
    <Heading level={5}>Heading 5</Heading>
    <Heading level={6}>Heading 6</Heading>
  </>
```

## Size

There are six relative sizes for the heading: `headingXLarge`, `headingLarge`, `headingMedium`, `headingSmall`, `headingXSmall` and `headingXXSmall`. By default each of the size-styles corresponds with the semantic HTML heading elements levels 1-6.  
However any of these size-styles can be applied to of the HTML elements using the `size` prop.

Furthermore, the `size` prop can also accept the values for the four different paragraph sizes: `bodyLarge`, `bodyMedium`, `bodySmall`, and `bodyXSmall`. These are sizes are also used with the <Link to="/api/paragraph">Paragraph</Link> component.

```.jsx
  <>
    <Heading level={2} size="headingXLarge">H2 styled as Heading XLarge</Heading>
    <Heading level={2} size="headingLarge">H2 styled as Heading Large</Heading>
    <Heading level={2} size="headingMedium">H2 styled as Heading Medium</Heading>
    <Heading level={2} size="headingSmall">H2 styled as Heading Small</Heading>
    <Heading level={2} size="headingXSmall">H2 styled as Heading XSmall</Heading>
    <Heading level={2} size="headingXXSmall">H2 styled as Heading XXSmall</Heading>

    <Heading level={2} size="bodyLarge">H2 styled as Body Large</Heading>
    <Heading level={2} size="bodyMedium">H2 styled as Body Medium</Heading>
    <Heading level={2} size="bodySmall">H2 styled as Body Small</Heading>
    <Heading level={2} size="bodyXSmall">H2 styled as Body XSmall</Heading>
  </>
```

## Variant

The `variant` prop can be used to set additional styles on a component. The `variant` can affect the font-size, or font-family. The default `variant` is `default`.

The `expressive` variant can be used in marketing contexts, which may require more variety than the default styles.

The `narrative` variant can be used for content-heavy contexts, such as an e-reader.

```.jsx
    <>
      <Heading variant="expressive" level={1}>Expressive Heading 1</Heading>
      <Heading variant="expressive" level={2}>Expressive Heading 2</Heading>
      <Heading variant="expressive" level={3}>Expressive Heading 3</Heading>
      <Heading variant="expressive" level={4}>Expressive Heading 4</Heading>
      <Heading variant="expressive" level={5}>Expressive Heading 5</Heading>
      <Heading variant="expressive" level={6}>Expressive Heading 6</Heading>
      <br/><br/>
      <Heading variant="narrative" level={1}>Narrative Heading 1</Heading>
      <Heading variant="narrative" level={2}>Narrative Heading 2</Heading>
      <Heading variant="narrative" level={3}>Narrative Heading 3</Heading>
      <Heading variant="narrative" level={4}>Narrative Heading 4</Heading>
      <Heading variant="narrative" level={5}>Narrative Heading 5</Heading>
      <Heading variant="narrative"  level={6}>Narrative Heading 6</Heading>
    </>
```

## Color

The `color` prop can be used to change the style or emphasis. It accepts the values: `success`, `danger` and `subdued`.

```.jsx
    <>
        <Heading color="success" level={2}>
            Success Color Heading
        </Heading>

        <Heading color="danger" level={2}>
            Danger Color Heading
        </Heading>

        <Heading color="subdued" level={2}>
            Subdued Color Heading
        </Heading>
    </>
```

## Inverse

The `isInverse` property is an optional boolean that changes the colors so the component can appear on a dark background.

```.jsx
  <div style={{ background : '#00263E', padding: '10px' }}>
    <Heading isInverse level={2}>Heading 2</Heading>
    <Heading isInverse level={2} color="subdued">Heading 2 Subdued</Heading>
  </div>
```

## No Margins

By default, the `Heading` component includes margins on the top and bottom. When the `noMargins` prop is applied, the element will have a margin value of 0.

```.jsx
  <>
    <Heading noMargins level={1}>Heading 1</Heading>
    <Heading noMargins level={2}>Heading 2</Heading>
    <Heading noMargins level={3}>Heading 3</Heading>
    <Heading noMargins level={4}>Heading 4</Heading>
    <Heading noMargins level={5}>Heading 5</Heading>
    <Heading noMargins level={6}>Heading 6</Heading>
  </>
```

## Forward Ref

Using React's `forwardRef` feature you can gain access to the reference of the internal heading.

```.jsx
  (props) => {
    const headingRef = React.useRef();

    return <div>
      <Heading
        ref={headingRef}
        level={1}>
        Heading to Be Focused
      </Heading>

      <Button
        onClick={() => {
          headingRef.current.tabIndex = -1
          headingRef.current.focus();
        }}>Click to focus heading
      </Button>
    </div>
  }
```

## Testing

Passing in the `testId` prop will add the `data-testid` attribute to the heading element for easier querying in tests.

```html
<h1 data-testid="test-id">Heading 1</h1>
```
