## Basic Usage

Radio buttons are form elements used to select one item from a set of mutuallty exclusive options.
See also: <Link to="/api/checkbox">Checkboxes</Link>, <Link to="/api/toggle">Toggle Switches</Link>, and the <Link to="/design/selection-controls">Design Guidelines</Link> for selection controls in general.

The `RadioGroup` component is used as the container for the set of options.  The `Radio` component is used for each of the options themselves.


```.jsx
  <RadioGroup labelText='Basic Usage' id='basicGroup' name='basic'>
    <Radio
        id='radio1'
        labelText='Option One Label'
        value='1' />

    <Radio
        id='radio2'
        labelText='Option Two Label is really long and can wrap to multiple lines lorem ipsum dolar sit amet is really long and can wrap to multiple lines'
        value='2' />
    </RadioGroup>
```

## Default Selected Value

To set a selected value by default, use the `value` prop on the `RadioGroup` to pass in the `value` of the option you wish to select.

```.jsx
  <RadioGroup labelText='Default Selected Value' id='defaultSelectedGroup' name='defaultSelected' value='selectedOption'>
    <Radio
        id='radioSelected'
        labelText='Selected Option'
        value='selectedOption' />

    <Radio
        id='radioUnselected'
        labelText='Unselected Option'
        value='unselectedOption' />
    </RadioGroup>
```

## Colors

By default, the radio buttons use the theme's primary color. But any other color may be passed in.

```.jsx
  <RadioGroup labelText='Colors' id='colorsGroup' name='colors'>
    <Radio
        id='colorRadio'
        labelText='Default Color'
        value='default' />
      <Radio
          color={magma.colors.success01}
          id='successColorRadio'
          labelText='Success Color'
          value='success' />
      <Radio
          color={magma.colors.danger}
          id='dangerColorRadio'
          labelText='Danger Color'
          value='danger' />
      <Radio
          color={magma.colors.pop02}
          id='popColorRadio'
          labelText='Pop Color'
          value='pop' />
    </RadioGroup>
```

## Inverse Colors

Inverse is an optional boolean prop, that may be passed in when radio buttons are to be displayed on a dark background.

```.jsx
  <RadioGroup
    style={{ background : '#00263E', color: '#fff', padding: '10px' }}
    labelText='Inverse Radio Buttons'
    id='inverseGroup'
    name='inverse'>
    <Radio
        id='inverseRadio'
        inverse
        labelText='Inverse'
        value='default' />
      <Radio
          color={magma.colors.success01}
          id='successInverseRadio'
          inverse
          labelText='Success Inverse'
          value='success' />
      <Radio
          color={magma.colors.danger}
          id='dangerInverseRadio'
          inverse
          labelText='Danger Inverse'
          value='danger' />
      <Radio
          color={magma.colors.pop02}
          id='popInverseRadio'

          value='pop' />
        <Radio
            disabled
            id='disabledInverseRadio'
            inverse
            labelText='Disabled Inverse'
            value='disabled' />
    </RadioGroup>
```

## Custom Styles

Custom styles can be passed into both the RadioGroup component and the Radio component. The `style` property will apply to the container.
Additional `labelStyle` and `inputStyle` properties are available to style the respective elements. Please use discretion when adding custom styles.

```.jsx
  <RadioGroup
    style={{ 'border': '2px solid yellow', 'padding': '10px'}}
    labelStyle={{ 'color': 'blue', 'textTransform': 'uppercase'}}
    labelText='Custom styles'
    id='customGroup'
    name='custom'>
    <Radio
        id='custom1'
        inputStyle={{ 'boxShadow': '3px 3px 3px rgba(0, 0 , 0, .6)'}}
        labelStyle={{ 'color': 'red', 'fontStyle': 'italic'}}
        labelText='Custom Styles'
        style={{ 'border': '1px dotted purple'}}
        value='1' />

    <Radio
        id='custom2'
        labelText='No Custom Styles'
        value='2' />
  </RadioGroup>
```

## Visually Hidden Label

Adding the `textVisuallyHidden` prop will make the label text available to screen readers, but will hide it visually.

```.jsx
  <RadioGroup
      labelText='Hidden Label'
      id='visuallyHiddenRadioGroup'
      name='visuallyHiddenRadioGroup'
      textVisuallyHidden>
        <Radio
            id='visuallyHiddenRadio1'
            labelText='Hidden Label'
            textVisuallyHidden />
        <Radio
            id='visuallyHiddenRadio2'
            labelText='Hidden Label'
            textVisuallyHidden />
    </RadioGroup>
```

## Forward Ref

Using React's `forwardRef` feature you can gain access to the reference of the internal radio.

```.jsx
  class Example extends React.Component {
      constructor(props) {
        super(props)
        this.radioRef = React.createRef();
      }

      render() {
        return <div>
        <RadioGroup labelText='ForwardRef Usage' id='forwardRefGroup' name='forwardRef'>
          <Radio
            ref={this.radioRef}
            id='forwardRefRadio1'
            labelText='Option One Label'
            value='1' />

          <Radio
            id='forwardRefRadio2'
            labelText='Option Two Label is really long and can wrap to multiple lines lorem ipsum dolar sit amet is really long and can wrap to multiple lines'
            value='2' />
        </RadioGroup>

        <Button
          onClick={() => {
            this.radioRef.current.checked = !this.radioRef.current.checked
          }}>Toggle option 1
        </Button>

         <Button
          color="secondary"
          onClick={() => {
            this.radioRef.current.focus()
          }}>Focus option 1
        </Button>
        </div>
      }
    }
```
